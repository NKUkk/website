WebCore渲染（Ⅰ）——基础
发表于 2021-02-01| 更新于 2021-02-03|WebCore
阅读量:|评论数:

最近要学一下WebCore，记录一下学习过程，方便查阅。本文英文原文为WebCore Rendering I – The Basics。

DOM树

网页被解析成称为文档对象模型（Document Object Model，简称DOM）的节点树。树中所有节点的基类为Node。
节点分为几个类别，其中和渲染有关的节点类型如下：

  • 文档（Document）：树的根终究是文档节点。文档节点也分为Document、HTMLDocument以及SVGDocument三种类型，其中第一种Document可应用于除SVG文档以外的所有XML文档。HTMLDocument继承于Document，但只可用于HTML文档。SVGDocument也继承于Document，但只可用于SVG文档。
  • 元素（Elements）：HTML或XML源代码中出现的所有标签都变成了元素。从渲染的角度来看，元素是具有标签名称的节点，可用于将其强制转换为可查询渲染器所需数据的特定子类。查看Element.h。
  • 文本（Text）：元素之间出现的原始文本将变成文本节点。文本节点存储此原始文本，并且渲染树可以查询该节点以获取其字符数据。查看Text.h。

渲染树

渲染的核心是渲染树。渲染树和DOM树非常相似，渲染树是一个对象树，其中每个对象都可以对应于文档、元素或文本节点。渲染树还可以包含没有相应DOM节点的其他对象。所有渲染树节点的基类RenderObject。每个DOM节点的renderObject对象可以通过节点的renderer()方法获得，即’RenderObject* renderer() const’。

创建渲染树

渲染器是通过DOM上的附加（attachment）过程创建的。解析文档或添加DOM节点时，将在DOM节点上调用一种名叫attach的方法来创建渲染器，即’void attach()’。attach方法会计算DOM节点的样式信息。如果该元素的display CSS属性设置为none或者该节点是具有display:none设置的元素的后代，则不会创建该节点的任何渲染器。节点的子类和CSS显示属性值一起用于确定节点制作哪种渲染器。
附加是一种自顶而下的递归操作，在任何后代创建其渲染器之前，其父节点始终要先完成渲染器的创建。

销毁渲染树

当从文档中删除DOM节点或文档被拆除时（例如它位于其中的选项卡或窗口被关闭），渲染器会被破坏，这时将在DOM节点调用一种名叫detach的方法来断开和破坏渲染器，即void detach()。
分离的过程是自下而上的递归操作，在父级销毁其渲染器之前，其后代节点都会先销毁自己的渲染器。

访问样式信息

在附加的过程中，DOM会查询CSS以获取元素的样式信息，获取到的信息会被存储在RenderStyle对象中。WebKit支持的每一个CSS属性都可以通过此对象进行查询。RenderStyle是引用计数的对象。如果DOM节点创建了渲染器，则它将使用渲染器上的setStyle方法，即void setStyle(RenderStyle)，将样式信息连接到该渲染器。渲染器会添加一个对样式的引用，该样式将一直保持到渲染器获得新的样式为止。每个RenderObject对象可以使用style方法，即RenderStyle style() const，访问该对象的存储在其RenderStyle里的样式信息。

CSS盒模型

RenderObject最主要的子类是RenderBox，即遵循CSS盒模型的对象。该子类对象包括所有拥有边框（borders）、填充（padding）、边距（margins）、宽度（width）和高度（height）的对象。
CSS 2.1规格的示意图说明了CSS盒子的各部分，下列方法可用来获得边框、边距、填充的宽度。除非是查看原始的样式信息，否则RenderStyle不应该使用，因为最终RenderObject得到的计算值会有很大不同（特别是表格，可以覆盖单元格的填充，单元格之间也可以有展开的边框）。

width()和height()方法获得包括边界在内的盒子宽度和高度。

客户盒子(client box)是盒子(box)除边框和滚动条之外但包括填充在内的区域。

内容盒子(content box)用来表示CSS盒子除边框和填充之外的区域。

当盒子有垂直或这水平滚动条时，它们放在填充和边框之间，滚动条的宽度和高度包括在client宽度和client高度中。滚动条不是内容盒子的组成部分。可滚动区域的尺寸和当前滚动的位置都可以从RenderObject获得。

盒子还有x和y坐标位置，这些位置都是相对于祖先节点的，它们负责决定盒子(box)放在何处。但存在一些例外，这也是渲染树让人难以理解的领域之一。
